# nosi base package set for the Raspberry Pi OS (arm64) headless image.
#
# Why this file exists: the x86 nosi variants install their baseline via
# cloud-init's `packages:` block in the per-variant .user file. The Pi image
# is NOT baked by booting + cloud-init; it is baked by loop-mounting the
# official Raspberry Pi OS image and running apply.sh in a chroot
# (cijoe/scripts/rpi_image_build.py). cloud-init never runs, so the chroot
# build is the Pi's delivery mechanism and owns the package install that the
# `packages:` block owns elsewhere. rpi_image_build.py reads this file
# (ignoring blank lines and `#` comments) and `apt-get install`s it inside
# the chroot before running apply.sh. 06-package-presence.sh still asserts the
# critical subset afterwards, so a bad/renamed name fails the bake loudly.
#
# Curated from cloudinit-headless-debian-13.user (RPi OS is Debian-family, so
# it resolves the same debian apt branch in every provision step). x86-only
# entries are dropped: linux-headers-amd64 (an operator who wants out-of-tree
# modules installs linux-headers-$(uname -r) on the Pi), linux-perf,
# intel-microcode / amd64-microcode (no arm equivalent), ovmf, qemu-system-x86,
# firmware-misc-nonfree (RPi OS already ships the Pi firmware + brcm wifi/bt
# blobs it needs). qemu-system-x86 -> qemu-system-arm so cijoe can still drive
# emulated guests.
